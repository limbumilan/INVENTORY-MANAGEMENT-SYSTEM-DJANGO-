8. Create Core Models
8.1 Product Model
class Product(models.Model):
    name = models.CharField(max_length=255)
    sku = models.CharField(max_length=100, unique=True)
    category = models.CharField(max_length=100)
    price = models.DecimalField(max_digits=10, decimal_places=2)
8.2 Stock Model
class Stock(models.Model):
    product = models.ForeignKey(Product, on_delete=models.CASCADE)
    quantity = models.IntegerField(default=0)
8.3 Stock Movement (IN/OUT tracking)
class StockMovement(models.Model):
    IN = "IN"
    OUT = "OUT"

    MOVEMENT_TYPE = [
        (IN, "Stock In"),
        (OUT, "Stock Out"),
    ]

    product = models.ForeignKey(Product, on_delete=models.CASCADE)
    quantity = models.IntegerField()
    movement_type = models.CharField(max_length=10, choices=MOVEMENT_TYPE)
    created_at = models.DateTimeField(auto_now_add=True)
8.4 Supplier Model
class Supplier(models.Model):
    name = models.CharField(max_length=255)
    phone = models.CharField(max_length=20)
    email = models.EmailField()

11. Stock Logic (Important)

Stock OUT
Decrease quantity

if stock.quantity >= qty:

12. Basic Views (CRUD)

Example Product Create:

from django.shortcuts import render, redirect
from .models import Product

def add_product(request):
    if request.method == "POST":
        Product.objects.create(
            name=request.POST['name'],
            sku=request.POST['sku'],
            price=request.POST['price']
        )
        return redirect('product_list')

Key Details in the Flow:
The Product Table serves as the central directory. Both of the other tables point back to it using its ID.

The Stock Table shares a strict One-to-One (1:1) relationship with the Product. Every single product has exactly one matching active stock counter shelf.

The StockMovement Table shares a One-to-Many (1:N) relationship with the Product. A single product can have thousands of individual movement logs over its lifetime, tracking every historical transaction.
